World Models

A literature landscape

Terminology & taxonomy · prediction patterns & objectives · inputs/outputs · notable models · training & inference cost.

One term, three communities

A world model predicts how an environment evolves — given a state (and optionally an action), it predicts future states, observations, and/or rewards — for planning, control, imagination, or simulation. The three communities mean different things by it:

CommunityPredictsPurposePrediction spaceScaleExamples
Model-based RLreward / value / next latentplan & learn a policylow-dim latenttiny (≤200M, 1 GPU)Dreamer, MuZero, IRIS, TD-MPC2
Generative simulationnext frame (pixels)interactive "neural game engine"pixels / image tokenslarge (0.5–14B, clusters)GameNGen, Genie, DIAMOND, Cosmos
Representation modelsfuture embeddingself-supervised features & planningabstract latent (no decoder)efficient (no pixel head)I-JEPA, V-JEPA 2, DINO-WM

Provenance of the term: Schmidhuber 1990 (recurrent model nets) → Ha & Schmidhuber 2018 (V–M–C; train a policy "in a dream") → LeCun 2022 (JEPA: predict in representation space) → 2024+ "neural game engine / world simulator". The same word now spans control, video generation, and representation learning.

Correction (verified): "prediction space" is binary, not three-way. Dreamer (RSSM) and IRIS predict latents trained with a pixel decoder (loss = reconstruction + KL); JEPA / DINO-WM predict latents with no decoder and no reconstruction. So "latent state" vs "representation" is a difference of objective (next slide), not a separate prediction space.

Objective functions: five families

Reconstruction (VAE / ELBO)PlaNet, Dreamer V1–V3
reconstruction + KL — learns a latent by regenerating observations: a decoder-anchored latent.
Latent self-predictionJEPA, V-JEPA, DINO-WM
‖ predict − stopgrad(target) ‖ in latent space — no decoder, no pixel loss (the "representation" route); needs collapse prevention (EMA target / VICReg).
Autoregressive token NLLIRIS, TWM, Genie, iVideoGPT
VQ-tokenize frames, then maximize −log p(next token | past, action) — dynamics as sequence modeling.
Diffusion / denoisingDIAMOND, GameNGen, Oasis, Cosmos
Denoise the next frame / latent conditioned on past frames + action. Highest visual fidelity; cost scales with denoising steps.
Value-equivalentMuZero, EfficientZero, TD-MPC2
Predict no pixels — only reward + value + policy (MuZero) or latent consistency + reward + value (TD-MPC2). The model need only be accurate for planning.

Inputs and outputs

Inputs

  • Observations — pixels (64×64 → 720p), proprioception, depth, multi-camera, or text.
  • Actions / control — given discrete or continuous (Dreamer, MuZero); latent / inferred without labels (Genie); keyboard + mouse (GameNGen, Oasis); text "world events" (Genie-3).
  • Context — a window of past frames / latents (GameNGen: last 64 frames + 64 actions).

Outputs (= the prediction target)

  • Next observation — RGB frames at a resolution & FPS (generative engines).
  • Next latent state — compact code for imagination + policy learning (Dreamer, IRIS).
  • Reward / value / policy — value-equivalent, no pixels (MuZero, TD-MPC2).
  • Future representation — an embedding to plan against (JEPA, DINO-WM).

Output cost ordering: reward/value < latent < representation < pixels. Cheaper outputs give cheaper models but less directly interactive rollouts.

Inference cost & efficiency levers

Inference regimeCostExamples
Amortized policycheapest — one forward pass; model used only in trainingDreamer, IRIS, Δ-IRIS
Search / planningheavy per step even if training is cheap; scales with #simulationsMuZero (MCTS 50–800), TD-MPC2 (MPPI 512), PlaNet (CEM)
Generative rolloutper-frame diffusion/AR; real-time only with few steps / distillationGameNGen 20 FPS, DIAMOND 10 FPS, Genie-3 24 FPS; Cosmos/GAIA batch

Cheap to train ≠ cheap to run: a value-equivalent model (MuZero) is small but does expensive tree search at every step.

Efficiency levers for a tiny world model

Latent-space dynamics (RSSM) instead of pixel prediction — Dreamer.
Discrete tokens + small transformer — IRIS, Δ-IRIS (4 tokens/frame).
No reconstruction (drop the decoder) — JEPA, DINO-WM.
Self-supervised / consistency loss for sample efficiency — EfficientZero.
Imagination + replay — train the policy on synthetic rollouts (Atari100k regime).
Distillation / synthetic data; SSM (Mamba) backbones for long context at near-linear cost.

Takeaways

  • One term, three communities — model-based RL, generative simulation, and representation models predict reward/value, pixels, and embeddings respectively.
  • The objective is the main design lever — reconstruction · latent self-prediction · AR-token · diffusion · value-equivalent. "Latent state" vs "representation" is an objective distinction, not a separate prediction space.
  • Cost spans ~6 orders of magnitude in compute and ~4 in parameters; and cheap to train does not imply cheap to run.
  • Capability is not monopolized by scale — the strongest sample-efficiency results (superhuman Atari100k, Minecraft diamond, real-robot planning) come from sub-200M, single-GPU models. The cost gap buys visual fidelity and open-domain generality.
  • Decision for this project — which lineage and benchmark to target, and which efficiency lever to push: no-decoder (JEPA), context-aware tokenization (Δ-IRIS), or SSM backbones.
